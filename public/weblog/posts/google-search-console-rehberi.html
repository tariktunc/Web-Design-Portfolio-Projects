Google Search Console: Kapsamlı Kullanım Rehberi

Google Search Console (GSC), web sitenizin Google arama sonuçlarındaki performansını izlemenize, indeksleme sorunlarını tespit etmenize ve arama görünürlüğünüzü optimize etmenize yardımcı olan ücretsiz bir Google aracıdır. Her web geliştiricisi ve SEO uzmanı için vazgeçilmez olan GSC'nin tüm özelliklerini bu kapsamlı rehberde ele alıyoruz.

Google Search Console'a Başlarken

GSC'yi kullanmaya başlamak için önce sitenizin sahipliğini doğrulamanız gerekir. Dört doğrulama yöntemi mevcuttur:

1. DNS Doğrulaması (Önerilen)

Domain mülkiyeti (property) oluşturmak için DNS doğrulaması en kapsamlı yöntemdir. Tüm alt alan adlarını ve protokolleri (HTTP/HTTPS) kapsar.

2. HTML Tag Doğrulaması

<!-- Ana sayfanızın <head> bölümüne ekleyin -->
<meta name="google-site-verification" content="DOGRULAMA_KODU_BURAYA">

<!-- Next.js App Router ile -->
// app/layout.tsx
export const metadata: Metadata = {
  verification: {
    google: 'DOGRULAMA_KODU_BURAYA',
  },
};

3. HTML Dosyası Yükleme

Google'ın verdiği HTML dosyasını sitenizin kök dizinine yükleyin.

4. Google Analytics veya Tag Manager

Zaten Google Analytics veya Tag Manager kullanıyorsanız, bu araçlar üzerinden otomatik doğrulama yapabilirsiniz.

İpucu: Domain mülkiyeti (domain property) oluşturmak, URL prefix mülkiyetinden daha avantajlıdır. Domain property, tüm alt alan adlarını (www, blog, m. vb.) ve her iki protokolü (HTTP/HTTPS) tek bir mülk altında toplar.

Performance Raporu

Performance raporu, sitenizin Google arama sonuçlarındaki performansını detaylı olarak gösterir. Dört temel metrik sunar:

  • Total Clicks: Arama sonuçlarından sitenize yapılan toplam tıklama sayısı.
  • Total Impressions: Sayfalarınızın arama sonuçlarında görüntülenme sayısı.
  • Average CTR: Tıklama oranı (Clicks / Impressions).
  • Average Position: Arama sonuçlarındaki ortalama sıralama konumu.

Performance Verilerini API ile Çekme

// Google Search Console API kullanımı
async function getSearchAnalytics(siteUrl: string, startDate: string, endDate: string) {
  const response = await fetch(
    `https://searchconsole.googleapis.com/webmasters/v3/sites/${encodeURIComponent(siteUrl)}/searchAnalytics/query`,
    {
      method: 'POST',
      headers: {
        'Authorization': `Bearer ${accessToken}`,
        'Content-Type': 'application/json',
      },
      body: JSON.stringify({
        startDate,
        endDate,
        dimensions: ['query', 'page'],
        rowLimit: 100,
        dimensionFilterGroups: [{
          filters: [{
            dimension: 'country',
            expression: 'tur',
          }],
        }],
      }),
    }
  );

  const data = await response.json();

  return data.rows?.map(row => ({
    query: row.keys[0],
    page: row.keys[1],
    clicks: row.clicks,
    impressions: row.impressions,
    ctr: (row.ctr * 100).toFixed(2) + '%',
    position: row.position.toFixed(1),
  }));
}

Page Indexing Raporu

Page Indexing raporu, Google'ın sitenizi nasıl indekslediğini gösterir. Sayfalarınızın indekslenme durumunu, hata ve uyarıları takip edebilirsiniz.

Yaygın indeksleme durumları:

  • Indexed: Sayfa başarıyla indekslenmiş.
  • Discovered - currently not indexed: Keşfedilmiş ama henüz indekslenmemiş.
  • Crawled - currently not indexed: Taranmış ama indekslenmemiş (kalite sorunu olabilir).
  • Excluded by 'noindex' tag: Noindex meta tag'i nedeniyle hariç tutulmuş.
  • Redirect: Sayfa yönlendirme yapıyor.
  • Duplicate without user-selected canonical: Canonical belirtilmemiş duplicate sayfa.
Uyarı: "Crawled - currently not indexed" durumu, Google'ın sayfanızı indekslemeye değer bulmadığı anlamına gelebilir. Bu durumda içerik kalitesini artırın, internal linking'i güçlendirin ve sayfanın benzersiz değer sunduğundan emin olun.

URL Inspection Aracı

Tek bir URL'nin Google'daki durumunu detaylı olarak incelemenize olanak tanır. Crawling, indexing, mobile usability ve rich results bilgilerini gösterir. Ayrıca bir URL'nin yeniden taranmasını talep edebilirsiniz (Request Indexing).

Sitemap Yönetimi

GSC üzerinden sitemap'lerinizi gönderebilir ve durumlarını izleyebilirsiniz:

  1. Sol menüden "Sitemaps" sekmesine gidin.
  2. Sitemap URL'nizi girin (örn. sitemap.xml).
  3. "Submit" butonuna tıklayın.
  4. Status, Discovered URLs ve Last Read bilgilerini takip edin.

Core Web Vitals Raporu

Core Web Vitals raporu, sitenizin sayfa deneyimi performansını gerçek kullanıcı verilerine (CrUX data) dayanarak gösterir. LCP, INP ve CLS metriklerini mobil ve masaüstü için ayrı ayrı raporlar.

// Core Web Vitals izleme (web-vitals kütüphanesi)
import { onLCP, onINP, onCLS } from 'web-vitals';

function sendToAnalytics(metric) {
  const body = JSON.stringify({
    name: metric.name,
    value: metric.value,
    rating: metric.rating, // 'good', 'needs-improvement', 'poor'
    delta: metric.delta,
    id: metric.id,
    page: window.location.pathname,
  });

  // Beacon API ile gönder (sayfa kapanırken bile çalışır)
  navigator.sendBeacon('/api/analytics/vitals', body);
}

onLCP(sendToAnalytics);
onINP(sendToAnalytics);
onCLS(sendToAnalytics);
Bilgi: GSC'deki Core Web Vitals verileri 28 günlük bir kayan pencere (rolling window) kullanır. Yaptığınız iyileştirmeler raporda görünene kadar en az 28 gün beklemeniz gerekir. Anlık test için Lighthouse veya PageSpeed Insights kullanın.

Düzenli GSC Kontrol Rutini

  • Günlük: Performance raporunda beklenmedik düşüş var mı kontrol edin.
  • Haftalık: Page Indexing raporunu inceleyin, yeni hataları düzeltin.
  • Aylık: Core Web Vitals, Manual Actions ve Security Issues kontrol edin.
  • Çeyreklik: Search Analytics verilerini indirip trend analizi yapın.

Sonuç

Google Search Console, web sitenizin Google'daki performansını ve sağlığını izlemek için en önemli araçtır. Performance raporu ile arama görünürlüğünüzü, Page Indexing ile indeksleme durumunuzu, Core Web Vitals ile sayfa deneyiminizi takip edebilirsiniz. GSC'yi düzenli olarak kontrol etmek, sorunları erken tespit etmenizi ve organik trafiğinizi sürdürülebilir şekilde artırmanızı sağlar.
