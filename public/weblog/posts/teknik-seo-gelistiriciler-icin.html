Teknik SEO: Web Geliştiriciler İçin Kapsamlı Rehber

Teknik SEO, bir web sitesinin arama motorları tarafından taranabilirliğini, indekslenmesini ve sıralanmasını doğrudan etkileyen altyapı optimizasyonlarının bütünüdür. İçerik kalitesi ne kadar yüksek olursa olsun, teknik altyapı sağlam değilse arama motorları sitenizi düzgün değerlendiremez. Bu rehberde, web geliştiricilerin bilmesi gereken tüm teknik SEO kavramlarını detaylı kod örnekleri ve uygulamalı adımlarla ele alacağız.

Teknik SEO Neden Önemli?

Google ve diğer arama motorları, web sayfalarını taramak (crawl), anlamak (parse) ve indekslemek (index) için karmaşık algoritmalar kullanır. Teknik SEO, bu süreçlerin sorunsuz işlemesini sağlar. Yavaş yüklenen sayfalar, bozuk bağlantılar, hatalı yönlendirmeler ve eksik meta bilgiler gibi teknik problemler, organik trafiğinizi ciddi ölçüde düşürebilir.

  • Crawl Budget Optimizasyonu: Arama motorlarının sitenize ayırdığı tarama bütçesini verimli kullanma.
  • Indexation Kontrolü: Hangi sayfaların indekslenmesi, hangilerinin hariç tutulması gerektiğini belirleme.
  • Rendering: JavaScript ağırlıklı sitelerin arama motorları tarafından doğru render edilmesini sağlama.
  • Site Mimarisi: URL yapısı, internal linking ve navigasyon hiyerarşisini optimize etme.

1. Sayfa Hızı ve Performance Optimizasyonu

Core Web Vitals, Google'ın sayfa deneyimini ölçmek için kullandığı temel metriklerdir. LCP (Largest Contentful Paint), INP (Interaction to Next Paint) ve CLS (Cumulative Layout Shift) değerlerini optimize etmek, sıralama faktörü olarak doğrudan etkilidir.

// next.config.js - Performans optimizasyonu
const nextConfig = {
  images: {
    formats: ['image/avif', 'image/webp'],
    deviceSizes: [640, 750, 828, 1080, 1200],
    minimumCacheTTL: 60 * 60 * 24 * 365, // 1 yıl
  },
  compress: true,
  poweredByHeader: false,
  reactStrictMode: true,
  experimental: {
    optimizeCss: true,
  },
  async headers() {
    return [
      {
        source: '/(.*)',
        headers: [
          {
            key: 'X-Content-Type-Options',
            value: 'nosniff',
          },
          {
            key: 'Cache-Control',
            value: 'public, max-age=31536000, immutable',
          },
        ],
      },
    ];
  },
};
İpucu: Statik asset'ler (resimler, fontlar, CSS/JS dosyaları) için uzun süreli cache header'ları kullanın. immutable direktifi, tarayıcının dosyayı yeniden doğrulama isteği göndermesini engeller.

2. HTML Head Yapısı ve Meta Tags

Doğru yapılandırılmış bir HTML head bölümü, arama motorlarına sayfanız hakkında temel bilgileri sağlar. Her sayfada benzersiz title ve description bulunmalıdır.

<!DOCTYPE html>
<html lang="tr">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1.0">
  <title>Teknik SEO Rehberi | Tarik Tunç</title>
  <meta name="description" content="Web geliştiriciler için teknik SEO rehberi. Crawling, indexing, site hızı ve yapısal veri optimizasyonu.">
  <meta name="robots" content="index, follow">
  <link rel="canonical" href="https://tariktunc.com/blog/teknik-seo-rehberi">

  <!-- Open Graph -->
  <meta property="og:type" content="article">
  <meta property="og:title" content="Teknik SEO: Web Geliştiriciler İçin Kapsamlı Rehber">
  <meta property="og:description" content="Crawling, indexing, site hızı ve yapısal veri optimizasyonu.">
  <meta property="og:image" content="https://tariktunc.com/images/teknik-seo.webp">

  <!-- Structured Data -->
  <script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": "Teknik SEO: Web Geliştiriciler İçin Kapsamlı Rehber",
    "author": {
      "@type": "Person",
      "name": "Tarik Tunç"
    },
    "datePublished": "2025-07-10"
  }
  </script>
</head>

3. Crawling ve Indexation Kontrolü

Arama motorlarının sitenizi nasıl taradığını kontrol etmek, teknik SEO'nun en kritik boyutlarından biridir. Robots.txt, meta robots tag'leri ve HTTP header'ları bu kontrol mekanizmalarının temelini oluşturur.

# robots.txt
User-agent: *
Allow: /
Disallow: /api/
Disallow: /admin/
Disallow: /_next/
Disallow: /private/

# Sitemap konumu
Sitemap: https://tariktunc.com/sitemap.xml

# Belirli botlar için kurallar
User-agent: GPTBot
Disallow: /

User-agent: Google-Extended
Disallow: /
Uyarı: robots.txt dosyası sayfaların indekslenmesini engellemez, sadece taranmasını kontrol eder. Bir sayfanın indekslenmemesini istiyorsanız meta robots noindex veya X-Robots-Tag HTTP header'ı kullanın.

4. URL Yapısı ve Yönlendirmeler

SEO dostu URL'ler kısa, açıklayıcı ve tutarlı olmalıdır. Gereksiz parametreler, session ID'ler ve büyük harfler URL kalitesini düşürür. Yönlendirme zincirleri (redirect chains) de crawl budget'ı israf eder.

  • Kısa ve açıklayıcı: /blog/teknik-seo-rehberi (iyi) vs /blog/post?id=123&cat=seo (kötü)
  • Küçük harf: Tüm URL'lerde küçük harf kullanın ve büyük harfleri 301 ile yönlendirin.
  • Trailing slash tutarlılığı: Ya her yerde kullanın ya da hiç kullanmayın.
  • Hyphens kullanın: Kelime ayırıcı olarak - kullanın, _ değil.

5. Internal Linking Stratejisi

Internal link'ler, hem kullanıcıların hem de arama motorlarının sitenizde gezinmesini sağlar. PageRank akışını yönlendirmek, önemli sayfaların otoritesini artırmak ve orphan page'leri (hiçbir yerden link almayan sayfalar) önlemek için bilinçli bir internal linking stratejisi şarttır.

// React bileşeni: Internal link audit
function InternalLinkChecker({ pages }) {
  const orphanPages = pages.filter(page => {
    const incomingLinks = pages.filter(p =>
      p.links && p.links.includes(page.url)
    );
    return incomingLinks.length === 0;
  });

  return (
    <div>
      <h3>Orphan Pages ({orphanPages.length})</h3>
      <ul>
        {orphanPages.map(page => (
          <li key={page.url}>{page.url} - {page.title}</li>
        ))}
      </ul>
    </div>
  );
}
Bilgi: Google'ın John Mueller, önemli sayfalara en az 2-3 farklı internal link verilmesini önerir. Breadcrumb navigasyonu ve ilgili yazılar bölümü, doğal internal linking oluşturmanın en etkili yollarıdır.

6. Structured Data ve Rich Results

Yapısal veri (structured data), arama motorlarına sayfa içeriğinizi daha iyi anlamalarını sağlayan standartlaştırılmış bir formattır. JSON-LD formatında eklenen schema markup'lar, arama sonuçlarında zengin snippet'ler (rich results) oluşturabilir. Article, FAQ, HowTo, BreadcrumbList ve Organization gibi schema tipleri en yaygın kullanılanlardır.

7. HTTPS ve Güvenlik

HTTPS, 2014'ten beri Google'ın resmi sıralama faktörüdür. Tüm HTTP trafiğini HTTPS'e 301 yönlendirme ile taşıyın. HSTS (HTTP Strict Transport Security) header'ı ile tarayıcıların her zaman güvenli bağlantı kullanmasını zorunlu kılın.

8. Mobile-First Indexing

Google, 2023 itibarıyla tüm web sitelerini mobile-first olarak indekslemektedir. Bu, sitenizin mobil versiyonunun indeksleme ve sıralama için birincil kaynak olarak kullanıldığı anlamına gelir. Responsive tasarım, mobil uyumlu font boyutları, dokunmatik hedeflerin yeterli büyüklükte olması ve viewport meta tag'inin doğru yapılandırılması kritik önem taşır.

9. JavaScript SEO Zorlukları

Modern SPA (Single Page Application) framework'leri (React, Vue, Angular) SEO açısından ek zorluklar getirir. Googlebot JavaScript'i render edebilir ancak bu süreç ikinci bir dalga (second wave) olarak gerçekleşir ve gecikmeye neden olabilir. SSR veya SSG kullanmak, JavaScript SEO sorunlarının en etkili çözümüdür.

10. Teknik SEO Audit Kontrol Listesi

  • Tüm sayfalar HTTPS üzerinden erişilebilir mi?
  • Robots.txt doğru yapılandırılmış mı?
  • XML Sitemap güncel ve doğru mu?
  • Canonical URL'ler doğru şekilde tanımlı mı?
  • 404 ve 500 hata sayfaları mevcut mu?
  • Core Web Vitals metrikleri yeşil bölgede mi?
  • Mobile uyumluluk testinden geçiyor mu?
  • Structured data hatasız mı (Schema Markup Validator)?
  • Broken link yok mu?
  • Redirect zincirleri 3'ten fazla adım içermiyor mu?
İpucu: Lighthouse, Screaming Frog, Ahrefs Site Audit ve Google Search Console gibi araçları düzenli olarak kullanarak teknik SEO sağlığınızı takip edin. Aylık audit rutini oluşturmak, sorunları erken tespit etmenizi sağlar.

Sonuç

Teknik SEO, web geliştirme sürecinin ayrılmaz bir parçasıdır. Sayfa hızından URL yapısına, crawling kontrolünden structured data'ya kadar her bileşen, sitenizin arama motorlarındaki görünürlüğünü doğrudan etkiler. Bu rehberdeki adımları uygulayarak teknik altyapınızı sağlamlaştırabilir, içeriklerinizin hak ettiği görünürlüğe kavuşmasını sağlayabilirsiniz. Teknik SEO bir defaya mahsus bir iş değil, sürekli izleme ve iyileştirme gerektiren bir süreçtir.
